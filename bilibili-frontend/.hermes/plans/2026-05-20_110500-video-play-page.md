# 视频播放页（Video Play Page）实施计划书

> 目标：将现有的原生 `<video>` 播放升级为 `DPlayer`，补齐播放页互动功能，打通「上传→转码→播放」完整链路。

---

## 一、当前现状与问题

### 1.1 已有基础

| 模块 | 现状 | 状态 |
|------|------|------|
| 路由 | `/video/:id` → `VideoDetailView.vue` | ✅ 已存在 |
| 详情 API | `GET /api/v1/videos/:id`（含播放量 +1） | ✅ 已存在 |
| 转码轮询 | `GET /api/v1/videos/:id/transcode`，3 秒轮询 | ✅ 已存在 |
| 视频列表 API | `GET /api/v1/videos?page=&size=` | ✅ 已存在 |
| 推荐列表 | `VideoDetailView` 右侧已调用 `getVideoList` | ✅ 已存在 |
| 类型定义 | `VideoDetail`、`VideoListItem` 等 | ✅ 已存在 |
| 播放器 | 原生 `<video>` 标签 | ⚠️ 需替换 |
| `VideoPlayer.vue` | 空文件 | ⚠️ 需填充 |

### 1.2 待解决问题

1. **播放器体验差**：原生 `<video>` 无快捷键、无倍速记忆、无画质切换、UI 与整体 cyber 风格不匹配
2. **互动功能缺失**：点赞、投币、收藏按钮只有展示数据，无交互和 API
3. **卡片跳转缺失**：首页 `VideoCard` 点击无反应，未跳转到播放页
4. **播放器未组件化**：播放逻辑直接写在 `VideoDetailView` 中，不利于复用和维护

---

## 二、总体方案

> **分阶段推进**：先做视觉骨架（播放器替换 + 页面布局），再补互动功能（点赞/投币/收藏），最后做细节打磨（分享、播放量精确上报、键盘快捷键）。

| 阶段 | 核心任务 | 预估工时 |
|------|----------|----------|
| **P0** | 集成 `DPlayer` 替换原生播放器；修复 `VideoCard` 跳转 | 2~3 小时 |
| **P1** | 点赞/投币/收藏 UI + 后端 CRUD 接口 | 3~4 小时 |
| **P2** | 分享功能、键盘快捷键、播放进度记忆（localStorage） | 2 小时 |

## 三、P0 阶段：播放器替换与页面骨架

### 3.1 技术选型：DPlayer

选用 `DPlayer` 而非 `Video.js` / `Plyr` 的理由：

- 轻量、API 简洁，中文文档友好
- 原生支持**弹幕**（为未来 P2 弹幕系统预留接口）
- 原生支持**多清晰度切换**（为未来多码率转码预留接口）
- UI 可深度自定义，适配 dark cyber 主题

安装命令：

```bash
npm install dplayer
npm install -D @types/dplayer
```

### 3.2 文件变更清单

| 文件路径 | 变更类型 | 说明 |
|----------|----------|------|
| `src/components/VideoPlayer.vue` | 重写 | 封装 `DPlayer`，支持传入 `videoUrl`、`coverUrl`、`options` |
| `src/views/VideoDetailView.vue` | 修改 | 替换 `<video>` 为 `<VideoPlayer>`，移除播放相关内联逻辑 |
| `src/components/VideoCard.vue` | 修改 | 添加 `@click="router.push(\`/video/\${video.id}\`)"` |
| `src/types/video.ts` | 新增字段 | `duration?: number`（可选，用于 DPlayer 预览进度条） |

### 3.3 VideoPlayer.vue 设计规格

```vue
<!-- VideoPlayer.vue 接口定义 -->
<script setup lang="ts">
interface Props {
  videoUrl: string      // 播放地址（转码后优先）
  coverUrl?: string     // 封面/poster
  videoId: number       // 用于 localStorage 记录播放进度
  autoplay?: boolean    // 默认 false
}
</script>
```

核心行为：

1. **转码中占位**：父组件传入 `transcode_status`，`VideoPlayer` 不负责此逻辑，`VideoDetailView` 在转码完成前仍展示现有转码占位 UI
2. **播放地址选择**：`VideoDetailView` 中判断 `transcoded_url || video_url`
3. **进度记忆**：`videoTime_${videoId}` 写入 `localStorage`，下次进入 `seekTo` 恢复（需在 `canplay` 事件后执行）
4. **暗色主题**：通过 `DPlayer` 的 `theme` 参数设为 `#00f0ff`，CSS 覆写控制条背景为 `#0a0a0f`

### 3.4 VideoDetailView.vue 调整点

1. **播放器替换**：
   ```vue
   <VideoPlayer
     v-else
     :video-url="getFullUrl(video.transcoded_url || video.video_url)"
     :cover-url="getFullUrl(video.cover_url)"
     :video-id="video.id"
   />
   ```

2. **互动按钮区域新增**（先做 UI 骨架，P1 再接入 API）：
   ```vue
   <div class="action-bar">
     <button class="action-btn" @click="onLike">
       <span class="action-icon">▲</span> 点赞 {{ formatViews(video.like_count) }}
     </button>
     <button class="action-btn" @click="onCoin">
       <span class="action-icon">◆</span> 投币
     </button>
     <button class="action-btn" @click="onFavorite">
       <span class="action-icon">★</span> 收藏
     </button>
     <button class="action-btn" @click="onShare">
       <span class="action-icon">⇧</span> 分享
     </button>
   </div>
   ```

3. **布局微调**：当前 `video-info` 区域在播放器下方，互动按钮插在 `video-info` 顶部，与 B站一致

### 3.5 VideoCard.vue 跳转修复

当前 `VideoCard` 使用 `VideoItem`（来自 `types/home.ts`），但 `VideoGrid` 实际传入的数据可能来自 `getVideoList`（返回 `VideoListItem`）。需确认数据流一致性。

**最简修复**：在 `VideoCard` 的 `div.video-card` 上添加 `@click` 事件：

```vue
<div class="video-card" @click="$router.push(`/video/${video.id}`)">
```

若 `VideoCard` 的 `video` prop 类型与列表接口不匹配，需要统一类型定义（优先使用 `types/video.ts` 中的 `VideoListItem`）。

## 四、P1 阶段：点赞 / 投币 / 收藏

### 4.1 数据库设计

```sql
-- video_likes 表（点赞）
CREATE TABLE video_likes (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    video_id BIGINT NOT NULL,
    created_at DATETIME DEFAULT NOW(),
    UNIQUE KEY uk_user_video (user_id, video_id)
);

-- video_coins 表（投币，限制最多 2 枚）
CREATE TABLE video_coins (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    video_id BIGINT NOT NULL,
    count INT NOT NULL DEFAULT 1 CHECK (count BETWEEN 1 AND 2),
    created_at DATETIME DEFAULT NOW(),
    UNIQUE KEY uk_user_video (user_id, video_id)
);

-- video_favorites 表（收藏）
CREATE TABLE video_favorites (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    video_id BIGINT NOT NULL,
    created_at DATETIME DEFAULT NOW(),
    UNIQUE KEY uk_user_video (user_id, video_id)
);
```

### 4.2 后端接口设计

| 方法 | 路径 | 功能 | 响应 |
|------|------|------|------|
| `POST` | `/api/v1/videos/:id/like` | 点赞/取消点赞 | `{ liked: boolean, like_count: number }` |
| `POST` | `/api/v1/videos/:id/coin` | 投币（1 或 2 枚） | `{ coin_count: number }` |
| `POST` | `/api/v1/videos/:id/favorite` | 收藏/取消收藏 | `{ favorited: boolean }` |
| `GET` | `/api/v1/videos/:id/actions` | 获取当前用户对视频的操作状态 | `{ liked, coined, favorited }` |

### 4.3 GORM Model（Go 后端）

在 `model/video.go` 同级新建 `model/video_interaction.go`：

```go
package model

type VideoLike struct {
    ID        uint64 `gorm:"primaryKey"`
    UserID    uint64 `gorm:"index:idx_user_video,unique"`
    VideoID   uint64 `gorm:"index:idx_user_video,unique"`
    CreatedAt int64  `gorm:"autoCreateTime"`
}
```

### 4.4 前端 API 层扩展

在 `src/api/video.ts` 新增：

```typescript
export async function toggleLike(id: number): Promise<{ liked: boolean; like_count: number }>
export async function giveCoin(id: number, count: number): Promise<{ coin_count: number }>
export async function toggleFavorite(id: number): Promise<{ favorited: boolean }>
export async function getVideoActions(id: number): Promise<{ liked: boolean; coined: boolean; favorited: boolean }>
```

### 4.5 互动按钮交互状态

在 `VideoDetailView` 中新增响应式状态：

```typescript
const actions = ref({ liked: false, coined: false, favorited: false })
```

登录用户才显示互动按钮，未登录点击跳转登录页（复用现有路由守卫逻辑）。

## 五、P2 阶段：细节打磨

### 5.1 分享功能

- 点击「分享」弹出 `navigator.clipboard.writeText(window.location.href)`
- Toast 提示「链接已复制」

### 5.2 键盘快捷键

通过 `DPlayer` 的 `hotkey` 配置启用：

| 按键 | 功能 |
|------|------|
| `Space` | 播放/暂停 |
| `←` / `→` | 快进/快退 5s |
| `↑` / `↓` | 音量 ±10% |
| `F` | 全屏 |
| `M` | 静音 |

### 5.3 播放进度持久化

- 监听 `DPlayer` 的 `timeupdate` 事件，每 5 秒写入 `localStorage`
- 视频播放至 90% 以上时清除记录（视为看完）
- 下次进入页面时，在 `canplay` 后 `dp.seek(savedTime)`，并显示「已为您定位到上次播放位置」提示

## 七、风险与待决策项

| 风险 | 影响 | 缓解方案 |
|------|------|----------|
| `@types/dplayer` 可能缺失或过时 | TypeScript 编译报错 | 声明 `declare module 'dplayer'` 兜底 |
| `DPlayer` 暗色主题 CSS 与现有样式冲突 | UI 异常 | 用 `::v-deep` 或全局 CSS 覆写 |
| 点赞/投币/收藏表增加后，视频列表查询变慢 | 首页加载慢 | 用 Redis 缓存计数，或联表查询时只取主表 |
| 播放进度 `localStorage` 无限增长 | 存储膨胀 | 只保留最近 20 条记录，超期清理 |
