# 计划书：首页视频列表真实化 + 个人投稿视频列表真实化

## 目标

1. 首页 `HomeView.vue` 从 `GET /api/v1/videos` 拉取真实视频数据，替代 mock
2. 个人空间「我的视频」`GET /api/v1/users/me/videos` 改为真实数据查询

## 当前状态

### 后端
- `controller/video.go`的`List`已是真实查询（`ListPublishedVideos`）
- `service/video.go`已有`ListMyVideos`方法（调用`dao.ListByUserID`）
- `controller/user.go`的`MyVideos`仍是 mock 数据
- `UserController`只有`userService`，无`videoService`

### 前端
- `HomeView.vue`仍在导入`mock/homeData`（`videoList`/全部/游戏分区）
- `api/video.ts`的`getVideoList` `USE_MOCK=false`，已走真实 API
- `PersonalView.vue`已调用`getMyVideos`，等后端返回真实数据
- 类型不兼容：后端返回`VideoListItem`（`cover_url`/`view_count`/`user`），前端`VideoItem`需要`coverGradient`/`duration`/`uploader`/`danmaku`/`date`

- 修改`controller.NewUserController`传入`videoService`

### Phase 2：前端—首页视频列表真实化

## 风险与备选方案

1. 类型不兼容：`VideoItem`原来用`coverGradient`（渐变色数组），现在需要用真实图片 URL。如果某些视频没有封面，需要 fallback 到渐变色或默认图。
2. 时间字段：后端返回`created_at`是 ISO 格式，前端需要转为相对时间（如"> 3h ago"）。
3. 无法确定`VideoCard`和`SectionBlock`的具体实现，需要实施时读取并调整。
