# 计划书：视频功能完善（时长 / 头像 / 点赞 / 收藏 / 历史）

## 目标

1. **视频时长**：后端添加 `duration` 字段，上传时通过 FFmpeg 提取
2. **头像加载修复**：视频详情页原作者头像显示真实图片而非文字首字母
3. **点赞功能**：高并发下使用 Redis 计数
4. **收藏功能**：数据库表 + 侧边栏入口 + 收藏列表页
5. **历史观看**：接入真实数据，限制 30 天 / 100 条

## 当前状态

- **Redis**：未接入，config 中无 Redis 配置
- **时长**：`model.Video` 无 `duration` 字段
- **头像**：后端返回 avatar 为相对路径 `/uploads/avatars/...`；前端 `VideoDetailView.vue` 的 up-avatar 用昵称首字母文字显示，未加载真实图片
- **点赞/收藏**：无数据库模型
- **历史**：`HistoryDao` 和 `VideoHistory` 模型已存在，但 `controller/user.go` 的 `History` 仍是 mock

### Phase 4：点赞功能（Redis + MySQL）

#### 4.1 数据库模型

```go
type VideoLike struct {
    BaseModel
    UserID  uint64 `gorm:"not null;uniqueIndex:idx_user_video"`
    VideoID uint64 `gorm:"not null;uniqueIndex:idx_user_video"`
    Status  int    `gorm:"default:1"` // 1=点赞 0=取消
}
```

文件：`main.go`
- `AutoMigrate` 加入 `&model.VideoLike{}`

#### 4.2 DAO 层

文件：`dao/like.go`（新建）
- `Create(like *model.VideoLike) error`

- `CountByVideo(videoID uint64) (int64, error)`

#### 4.3 Service 层（Redis 缓存 + 定时同步）

设计思路：
- 用户点赞 → 写 Redis `video:likes:count:{videoID}` INCR
- 用户取消 → 写 Redis `video:likes:count:{videoID}` DECR
- 同时记录 `video:likes:pending:{videoID}` Set，表示该视频有点赞变动待同步
- 用户是否点赞 → 查 Redis `video:likes:user:{userID}` Hash `{videoID: 1/0}`
- 定时任务（或后台 goroutine）每 30 秒将 Redis 中的点赞数同步回 MySQL `videos.like_count`

文件：`service/like.go`（新建）
- `LikeVideo(userID, videoID uint64) error`
- `UnlikeVideo(userID, videoID uint64) error`
- `IsLiked(userID, videoID uint64) (bool, error)`
- `GetLikeCount(videoID uint64) (int64, error)`
- `SyncLikesToDB()` 定时同步

#### 4.4 Controller 层

文件：`controller/like.go`（新建）
- `POST /api/v1/videos/:id/like` — 点赞/取消点赞（toggle）
- `GET /api/v1/videos/:id/like/status` — 查询当前用户是否点赞 + 总点赞数

#### 4.5 前端

文件：`bilibili-frontend/src/api/video.ts`

文件：`bilibili-frontend/src/views/VideoDetailView.vue`

### Phase 6：历史观看真实化

设计思路：
- 用户打开视频播放页 → 后端 `Detail` 接口记录播放历史
- 历史记录去重：同一视频再次观看只更新 `updated_at`
- 限制：只保留最近 30 天 / 最多 100 条
- 超出时自动清理最旧记录

文件：`controller/video.go`
- `Detail` 方法中加入 `videoService.RecordHistory(userID, videoID)`

文件：`service/video.go`
- 新增 `RecordHistory(userID, videoID uint64) error`
- 逻辑：
  1. 查询是否已有记录 → 有则更新 `updated_at`
  2. 无则插入新记录
  3. 检查总数是否 > 100，是则删除最旧的
  4. 删除超过 30 天的记录

文件：`controller/user.go`
- `History` 改为调用 `videoService.GetHistory(userID, page, size)` 真实查询
- 返回字段：`id`(video_id), `title`, `cover_url`, `views`, `watched_at`

文件：`bilibili-frontend/src/views/PersonalView.vue`
- 历史 Tab 已调用 `getHistory()`，后端改为真实数据后自动生效

## 风险与备选方案

1. **Redis 连接失败**：fallback 到直接写 MySQL，不阻塞点赞功能
2. **FFmpeg 提取时长失败**：设置默认 `duration=0`，不阻断上传流程
3. **点赞同步延迟**：Redis 到 MySQL 有 30 秒延迟，前端显示 Redis 实时数
4. **Phase 过多**：如果一次对话无法完成全部，优先完成 1/2/3（时长+头像+点赞）
