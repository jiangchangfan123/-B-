# 计划书：视频功能打磨（历史记录/封面/编辑）

## 目标

1. **历史记录**：显示真实封面，点击跳转到视频播放页
2. **已投稿视频**：显示真实封面
3. **视频编辑**：已投稿视频支持编辑（标题/简介/封面/视频文件）

---

## 执行阶段

### Phase 1：后端 — 视频更新接口

新建 `controller/video.go` 中新增 `Update` 方法：
- `PUT /api/v1/videos/:id` — 更新视频信息
- 支持修改：title, description, category, cover（可选）, video（可选）
- 校验：只有视频所有者才能编辑
- 如果上传了新视频文件，重新提取时长、重新发送转码任务

`service/video.go` 新增 `UpdateVideo`：
- 更新 title/description/category
- 如有新封面，上传到 MinIO 并更新 CoverURL
- 如有新视频，上传到 MinIO、提取时长、发送转码任务

`dao/video.go` 新增 `Update`：
- GORM Save 更新

---

### Phase 2：前端 — 历史记录封面+跳转

`PersonalView.vue`：
- `history-item` 添加 `@click="router.push(`/video/${item.id}`)"`
- 封面从文字占位符改为 `<img :src="getFullUrl(item.cover_url)" />`
- 添加封面加载失败的 fallback

---

### Phase 3：前端 — 已投稿视频封面

`PersonalView.vue`：
- `video-cover-placeholder` 从文字改为 `<img>`
- 使用 `getFullUrl(video.cover_url)`
- 失败 fallback 到 category 文字

---

### Phase 4：前端 — 编辑功能

`router/index.ts`：
- 新增 `/upload/:id` 路由（编辑模式）
- `/upload` 保持新建模式

`UploadView.vue`：
- 通过 `useRoute` 获取 `:id` 参数
- 如果有 id，进入编辑模式：
  1. `onMounted` 时调用 `getVideoDetail(id)` 加载原数据
  2. 填充表单（title, description, category）
  3. 显示当前封面预览
  4. 提交时调用 `PUT /api/v1/videos/:id`
- 新建模式保持原有逻辑

`api/video.ts`：
- 新增 `updateVideo(id, formData)`
- 新增 `getVideoDetail`（已有）

`PersonalView.vue`：
- 编辑按钮点击：`router.push(`/upload/${video.id}`)`
- 删除按钮保持原有功能

---

## 风险

- 编辑时上传新视频文件会触发重新转码，可能需要等待
- 封面/视频文件可选，不传则保留原文件
