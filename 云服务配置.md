# 仿B站项目 - 云服务器基础设施部署（仅 MySQL/Redis/MinIO/RabbitMQ）

> 适用场景：前后端代码在**本地开发**，云服务器只跑数据库/缓存/存储/队列等基础设施。

## 📁 云服务器上的文件

在你的云服务器上创建项目目录，放入以下文件：

```
infra/
├── docker-compose.yml      # 基础设施编排
├── .env                    # 密码配置
└── init/                   # (可选) MySQL 初始化 SQL 脚本
    └── 01-init.sql
```

## 🚀 云服务器一键启动

```bash
# 进入目录
cd infra

# 启动全部基础设施
docker compose up -d

# 查看状态
docker compose ps

# 查看日志
docker compose logs -f
```

启动成功后，云服务器会暴露以下端口：

| 服务 | 端口 | 管理面板/连接方式 | 默认账号 |
|------|------|-------------------|----------|
| MySQL | 3306 | `mysql -h 你的IP -P 3306 -u bilibili -p` | bilibili / bilibili123 |
| Redis | 6379 | `redis-cli -h 你的IP -a bilibili123` | 密码 bilibili123 |
| MinIO API | 9000 | SDK/直传使用 | minio / minio123456 |
| MinIO 控制台 | 9001 | `http://你的IP:9001` | minio / minio123456 |
| RabbitMQ | 5672 | 业务连接端口 | bilibili / bilibili123 |
| RabbitMQ 管理台 | 15672 | `http://你的IP:15672` | bilibili / bilibili123 |

## 💻 本地开发连接配置

### 1. 本地后端（Go）

把 `backend-env-local.example` 的内容复制到你本地后端项目的 `.env` 文件，修改 `{你的云服务器IP}` 为实际 IP。

关键配置示例：
```env
DB_HOST=123.45.67.89
REDIS_HOST=123.45.67.89
MINIO_ENDPOINT=123.45.67.89:9000
RABBITMQ_HOST=123.45.67.89
```

> 如果后端也在本地跑，本地启动命令就是普通的 `go run ./cmd/server`。

### 2. 本地前端（Vue 3 + Vite）

把 `frontend-env-local.example` 的内容复制为本地前端项目根目录的 `.env.local`：

```env
VITE_API_BASE_URL=http://123.45.67.89:8080/api/v1
VITE_WS_URL=ws://123.45.67.89:8080/ws
VITE_MINIO_ENDPOINT=http://123.45.67.89:9000
```

> 注意：这里假设你的**后端也部署在云服务器**（或内网穿透）。如果后端也在本地跑，API 地址写 `http://localhost:8080/api/v1` 即可，只有数据库等基础设施走云服务器。

## 🔒 安全加固（非常重要！）

你的数据库/Redis 端口暴露在公网上，**必须**做好安全防护：

### 方案 A：安全组/防火墙限制 IP（推荐）

在云服务器控制台的安全组规则中，限制以下端口**只允许你的本地 IP 访问**：
- 3306 (MySQL)
- 6379 (Redis)
- 5672 (RabbitMQ)
- 15672 (RabbitMQ 管理台)
- 9000/9001 (MinIO，如果不需要公网直传也可以限制)

示例（阿里云安全组规则）：
```
协议: TCP
端口: 3306
授权对象: 你的本地公网IP/32   ← 不是 0.0.0.0/0！
```

### 方案 B：SSH 隧道（更安全，不用暴露端口）

不在安全组开放任何数据库端口，通过 SSH 隧道连接：

```bash
# 在本地终端建立隧道，把云服务器的 3306 映射到本地 3306
ssh -N -L 3306:localhost:3306 root@你的云服务器IP

# 然后本地后端配置
DB_HOST=localhost
DB_PORT=3306
```

同理可以映射 Redis、RabbitMQ 等端口。

### 方案 C：修改默认密码（最低要求）

编辑云服务器上的 `.env` 文件，把所有默认密码改成强密码：
```bash
MYSQL_ROOT_PASSWORD=你的强密码
MYSQL_PASSWORD=你的强密码
REDIS_PASSWORD=你的强密码
MINIO_ROOT_PASSWORD=你的强密码
RABBITMQ_PASSWORD=你的强密码
```

修改后重启：
```bash
docker compose down
docker compose up -d
```

## 🛠️ 常用运维命令

```bash
# 查看所有服务状态
docker compose ps

# 查看 MySQL 日志
docker compose logs -f mysql

# 进入 MySQL 容器执行 SQL
docker exec -it bilibili-mysql mysql -ubilibili -p

# 进入 Redis 容器
docker exec -it bilibili-redis redis-cli -a bilibili123

# 停止所有基础设施
docker compose down

# 停止并清空所有数据（⚠️ 会删除数据库！）
docker compose down -v
```

## ⚠️ 注意事项

1. **不要暴露 3306/6379 到公网**：除非你做了 IP 白名单，否则很容易被扫描攻击。
2. **MinIO Bucket 初始化**：首次启动后访问 `http://你的IP:9001`，手动创建 `videos`、`covers`、`avatars` 三个 bucket，并设置访问策略为 `public` 或按需配置。
3. **MySQL 初始化**：如果已有建表 SQL，可以放到 `init/01-init.sql`，首次启动时 MySQL 会自动执行。
4. **网络延迟**：本地连接云服务器数据库会有 ~20-50ms 延迟，开发时几乎无感，但大量查询时注意优化。
