请基于以下修改后的需求，编写个人中心（PersonalView）页面，与之前的主页/登录页保持完全统一的深空科技风。

一、数据库变更

`users` 表确保以下字段存在：
- `avatar`：VARCHAR(255)，默认空字符串，头像 URL
- `sign`：VARCHAR(200)，默认空字符串，个性签名
- `email`：VARCHAR(128)，唯一索引

新增表（简化版播放历史，不记录秒级进度）：
- `video_history`：id（自增）、user_id（关联 users）、video_id（关联 videos）、created_at（观看时间）。联合索引 `(user_id, created_at)`。
- 不存 progress 字段，不实现断点续播，只记录"什么时候看过"。

二、后端 API 设计（全部需要 JWT 认证）

1. 获取当前登录用户信息
- GET `/api/v1/users/me`
- 返回：`id, username, email, avatar, sign, role, coins, created_at`
- 严禁返回 password

2. 更新个人资料
- PUT `/api/v1/users/me`
- 请求体：`{ sign?: string, email?: string }`
- 校验：sign 最长 200 字符，email 符合邮箱格式
- 用户名不可修改

3. 修改密码
- PUT `/api/v1/users/me/password`
- 请求体：`{ oldPassword: string, newPassword: string }`
- 校验：newPassword 6-30 位
- 业务逻辑：bcrypt 比对旧密码 → bcrypt 哈希新密码 → 更新数据库 → 返回成功
- 错误码：40001（参数错误）、40002（旧密码错误）
- 注意：不需要密码强度条，前端不做任何强度校验和可视化，只做空值和长度校验

4. 头像上传
- POST `/api/v1/users/me/avatar`
- `multipart/form-data`，字段名 `file`
- 限制：jpg/png/webp，最大 5MB
- 业务逻辑：接收文件 → UUID 重命名 → 上传到 MinIO 或本地 `uploads/avatars/` → 返回可访问 URL → 更新 users.avatar
- 返回：`{ avatar: "http://xxx/avatar/xxx.jpg" }`

5. 获取我的视频列表
- GET `/api/v1/users/me/videos?page=1&size=20`
- 返回视频列表（标题、封面、播放量、状态、创建时间），按 created_at 倒序分页

6. 获取播放历史（简化版）
- GET `/api/v1/users/me/history?page=1&size=20`
- 返回视频列表 + 观看时间（`watched_at` 字段，如 `"2025-05-19T14:30:00Z"`）
- 不返回 progress、不返回百分比、不实现断点续播

三、前端页面设计：个人中心（PersonalView）

延续深空科技风，使用标准三栏布局（顶部导航栏 + 左侧边栏 + 主体内容区）。

左侧个人信息卡片（280px）
- 背景 `#0f1117`，边框 1px `rgba(0,240,255,0.1)`，圆角 4px。
- 头像区：120×120 方形（直角），边框 2px 霓虹青 `#00f0ff`，带发光。Hover 显示半透明遮罩层 + 文字 `> UPLOAD`，点击触发文件选择。上传中显示扫描线扫过动画。
- 用户名：等宽字体，冷白 `#e4e5eb`，20px。下方显示用户 ID（如 `ID: 10086`，等宽字体，暗灰 `#5a5d6e`）。
- 个性签名：暗灰 `#8b8fa3`，14px，最多 2 行，超出省略号。未设置显示 `> 暂无签名...`。
- 数据栏：横向排列，等宽字体，暗灰，如 `▶ 12 投稿` `◆ 3.4K 播放` `■ 128 粉丝`。
- 操作按钮：`[ EDIT PROFILE ]` 电光紫按钮，hover 变霓虹青，带扫描线动画。

右侧 Tab 导航
选项：`// 资料设置 [SETTINGS]`、`// 我的视频 [ARCHIVE]`、`// 播放历史 [HISTORY]`、`// 账号安全 [SECURITY]`。
- 未选中：暗灰 `#5a5d6e`
- 选中：霓虹青 `#00f0ff`，底部 1px 青色实线 + `■` 方块指示器
- 切换瞬时响应（0.1s）

四、前端状态与交互

Pinia 用户 Store 扩展
- `updateProfile(data)`：调 PUT `/api/v1/users/me`，成功后更新本地 `userInfo`。
- `updateAvatar(file)`：调 POST `/api/v1/users/me/avatar`，FormData 上传，成功后更新 `userInfo.avatar`。
- `updatePassword(oldPwd, newPwd)`：调密码接口，成功后自动 `logout()` 并跳登录页。
- `fetchMyVideos(page)`：调我的视频接口，维护 `myVideos` 列表。
- `fetchHistory(page)`：调播放历史接口，维护 `historyList`（只含视频信息和观看时间，无 progress）。

头像上传交互
- 点击头像或 `> UPLOAD` 遮罩 → 唤起隐藏 `<input type="file" accept="image/*" hidden>`。
- 选择文件后前端先预览（FileReader base64 临时显示），同时调接口上传。
- 上传中头像显示扫描线动画，成功后替换为返回 URL。
- 限制 5MB，超限提示 `> ERROR: FILE_TOO_LARGE`。

路由
- 路径：`/personal`，命名路由 `personal`。
- 需要登录，路由守卫拦截未登录用户跳 `/login?redirect=/personal`。
